## 一、设计概述

### 1.1 设计目标

本 UI 设计文档定义了会话流程规则挖掘系统的用户界面交互规范，旨在为用户提供直观、高效的操作体验，支持从需求输入到配置输出的完整流程。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **简洁直观** | 界面布局清晰，操作流程自然，降低用户学习成本 |
| **状态可见** | 实时反馈系统处理状态，让用户了解当前进度 |
| **容错友好** | 支持跳过可选步骤，提供清晰的错误提示和恢复机制 |
| **信息分层** | 复杂信息通过选项卡、卡片等方式分层展示，避免信息过载 |

### 1.3 界面流程总览

#### 2.1.3 交互元素说明

| 元素 | 类型 | 说明 |
|------|------|------|
| **标题** | 文本 | "让AI为您起草流程"，24px，加粗，居中 |
| **需求输入框** | 多行文本 | 占位提示文本，最小高度 160px，支持多行输入 |
| **文件上传区** | 上传组件 | 可选，支持拖拽和点击上传，仅接受 JSON 格式 |
| **分类标签** | 单选按钮组 | 横向排列，默认选中"示例"，用于筛选下方示例 |
| **示例列表** | 列表+按钮 | 每个示例右侧有复制按钮，点击后自动填充到输入框 |
| **步骤指示器** | 文本 | 显示当前步骤位置 "步骤 1 / 5" |
| **继续按钮** | 主按钮 | 初始禁用状态，输入框有内容后启用 |

#### 2.1.4 交互规则

1. **输入验证**：
   - 需求输入框为空时，继续按钮禁用
   - 输入框有内容后，继续按钮启用

2. **文件上传**：
   - 支持拖拽上传和点击选择文件
   - 仅接受 JSON 格式文件
   - 上传成功后显示文件名和大小
   - 提供删除按钮可移除已上传文件

3. **示例填充**：
   - 点击示例右侧的 ↑ 按钮，将该示例文本填充到输入框
   - 填充后光标定位到输入框末尾

4. **分类筛选**：
   - 切换分类标签时，下方示例列表动态更新
   - 每个分类预置 3-5 个典型示例

#### 2.2.3 交互元素说明

| 元素 | 类型 | 说明 |
|------|------|------|
| **加载动画** | 动画 | 旋转的加载图标，表示系统正在处理 |
| **状态标题** | 文本 | 动态更新的处理状态描述 |
| **进度条** | 进度组件 | 显示整体处理进度百分比 |

2. **状态映射**：
   - 后端返回的状态码映射为友好的中文描述
   - 每个子任务完成时，对应项标记为 ✓

3. **完成跳转**：
   - 处理完成后自动跳转到界面3（澄清界面）
   - 跳转前显示 "处理完成，准备进入澄清环节..." 提示

#### 2.5.3 交互元素说明

**顶部操作栏**：

| 元素 | 类型 | 说明 |
|------|------|------|
| **标题** | 文本 | "Agent 配置生成完成" |
| **导出配置按钮** | 主按钮 | 下载完整的 Agent 配置包 (ZIP 格式) |
| **重新生成按钮** | 次按钮 | 返回界面 1，重新开始流程 |

**选项卡导航栏**：

| 选项卡 | 说明 |
|--------|------|
| **流程** | 展示生成的业务流程图 |
| **规则** | 以卡片形式展示全局规则和流程规则 |
| **术语** | 以卡片形式展示专业术语定义 |
| **工具** | 以卡片形式展示工具定义和参数 |
| **固定话术** | 以卡片形式展示预设回复话术模板 |

**流程视图内容**：

| 元素 | 类型 | 说明 |
|------|------|------|
| **流程列表** | 侧边栏 | 左侧显示所有生成的流程名称，点击切换 |
| **流程图** | 图形组件 | 右侧显示选中流程的可视化流程图 |
| **节点交互** | 交互元素 | 支持点击节点查看详情、缩放、拖拽 |

**规则/术语/工具/话术视图内容**：

| 元素 | 类型 | 说明 |
|------|------|------|
| **卡片列表** | 卡片组件 | 垂直排列的卡片，每个卡片展示一个规则/术语/工具/话术 |
| **卡片内容** | 文本 | 显示名称、ID、条件/定义/话术内容、动作/参数等关键信息 |
| **展开/折叠** | 交互 | 点击卡片可展开查看完整详情 |

**固定话术视图特有元素**：

| 元素 | 类型 | 说明 |
|------|------|------|
| **话术内容** | 文本块 | 显示预设的回复话术文本，支持变量占位符 |
| **关联规则** | 标签 | 显示该话术绑定的规则ID |
| **变量占位符** | 标签组 | 显示话术中可替换的变量，如 `{user_name}` |
| **复制按钮** | 图标按钮 | 点击复制话术内容到剪贴板 |

#### 2.5.4 交互规则

1. **选项卡切换**：
   - 点击选项卡标题切换视图
   - 切换时保持当前选中状态（如流程视图中选中的流程）

2. **流程视图交互**：
   - 左侧流程列表单选，选中项高亮显示
   - 右侧流程图支持鼠标滚轮缩放
   - 支持拖拽平移流程图
   - 点击节点显示节点详情弹窗

3. **卡片视图交互**：
   - 卡片默认显示摘要信息
   - 点击卡片展开显示完整详情
   - 支持同时展开多个卡片

4. **导出功能**：
   - 点击"导出配置"按钮下载 ZIP 文件
   - ZIP 包含完整的 Agent 配置目录结构
   - 下载时显示进度提示

5. **重新生成**：
   - 点击"重新生成"返回界面1
   - 保留上次输入的需求描述（可选）

## 五、异常处理设计

### 5.1 错误提示

| 场景 | 提示方式 | 内容 |
|------|----------|------|
| **网络错误** | 弹窗/Toast | "网络连接异常，请检查网络后重试" |
| **处理超时** | 弹窗 | "处理时间过长，请稍后查看结果或重新提交" |
| **文件格式错误** | 输入框下方提示 | "请上传 JSON 格式的文件" |
| **生成失败** | 弹窗 | "配置生成失败，错误信息：xxx，请重试" |

### 5.2 重试机制

1. **状态轮询失败**：
   - 自动重试 3 次
   - 失败后显示错误提示和"重试"按钮

2. **生成失败**：
   - 显示错误详情
   - 提供"重新生成"按钮返回界面1
   - 保留用户输入便于修改后重试
