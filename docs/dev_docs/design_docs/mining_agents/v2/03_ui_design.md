**版本**: v2.0  
**创建日期**: 2026-03-30  
**文档类型**: UI/UX 设计文档 (基于 8 步法流程重构)  
**关联设计文档**: `06_design_optimization_proposal.md`

## 一、改版概述

### 1.1 改版背景

原 UI 设计 (v1.0) 基于 5 步法流程，无法支撑新的 8 步法 SOP 挖掘流程。为匹配 06 设计文档中的架构优化，需对 UI 进行全面重构。

**核心变化**:
| 维度 | 旧方案 (v1.0) | 新方案 (v2.0) |
|------|-------------|-------------|
| **流程步骤** | 5 步 (Step1-5) | 8 步 (Step1-8) |
| **并行展示** | 仅 Step3-5 并行 | Step3-4 并行、Step5 高度并行、Step6 部分并行 |
| **人工介入点** | 仅 Step1 澄清 | Step1 澄清 + Step2 目标对齐确认 |
| **结果展示** | 单层级流程 | 3 层 SOP 架构 (主干/分支/子弹) |
| **状态管理** | 简单进度条 | 多维度并发状态监控 |

### 1.2 设计目标

1. **流程适配**: 完整支持 8 步法 SOP 挖掘流程
2. **并发可视化**: 直观展示 Step 5 的 N 节点并行执行状态
3. **分层展示**: 支持 3 层 SOP 架构的浏览和编辑
4. **过程可追溯**: 展示辩论记录、REACT 历史、Deep Research 报告
5. **灵活配置**: 支持跳过可选步骤、调整并发参数

### 1.3 设计原则

| 原则 | 说明 | 实施要点 |
|------|------|---------|
| **流程透明化** | 让用户清晰了解 8 步法的执行流程 | 步骤导航、状态实时反馈 |
| **并发可视化** | 将复杂的并发逻辑以图形化方式呈现 | 进度分组、子任务展开/折叠 |
| **信息分层** | 复杂信息通过选项卡、树形结构分层展示 | 主 SOP/分支 SOP/子弹 SOP 分层 |
| **灵活可控** | 支持用户干预关键决策点 | 人工确认环节、参数调整 |
| **过程可追溯** | 完整保存并展示执行历史 | 辩论记录、REACT 对话、研究报告 |

**关键设计决策**:
- ✅ **界面 4 为统一等待界面**: 将 Step 2-8 的所有处理流程整合到一个界面中
- ✅ **减少界面跳转**: 用户无需在多个界面间切换，一站式查看所有步骤进度
- ✅ **全局视角**: 顶部步骤导航让用户清晰了解整体流程
- ✅ **信息密度更高**: 同时展示已完成、进行中、待开始的步骤状态

### 3.4 界面 4：统一处理等待界面 (核心重构)

#### 3.4.1 功能定位

**统一处理等待界面**是整个系统的核心监控界面，覆盖 Step 2-8 的所有处理流程。通过动态步骤导航、模块化进度展示和实时状态反馈，让用户清晰了解整个处理过程。

**核心特性**:
- ✅ **动态步骤导航**: 顶部显示当前步骤 (Step 2-8),自动高亮正在执行的步骤
- ✅ **模块化进度卡**: 每个步骤细分为多个子任务模块，独立展示进度
- ✅ **实时状态刷新**: 每 3 秒轮询后端状态，支持 WebSocket 推送
- ✅ **关键决策点**: Step 2 完成后暂停，弹出确认对话框 (支持跳过)
- ✅ **并发可视化**: Step 5 的 N 个节点并行执行状态以分组形式展示
- ✅ **过程可追溯**: 每个步骤完成后提供详情查看入口 (辩论记录、REACT 历史等)

#### 3.4.3 交互规则 (通用 + 步骤特定)

**A. 通用交互规则** (适用于所有步骤):

1. **步骤导航点击**:

   - 点击正在执行的步骤 (⟳) 可展开/折叠该步骤的子任务
   - 未开始的步骤 (○) 不可点击

   - 每个步骤完成后，"[查看详情 ▶]"按钮可用
   - 弹出模态框展示该步骤的完整输出和过程数据
   - 支持下载中间产物文件

3. **实时刷新**:
   - 默认每 3 秒轮询一次后端状态
   - 支持手动刷新按钮
   - 状态变化时有平滑过渡动画

4. **暂停/恢复**:
   - 点击"暂停执行"可暂停所有步骤
   - 暂停后显示"继续执行"按钮
